# Декомпозиция DR-A · Инструмент 4: Entity-Relationship · Задача 1

**Инструмент:** Entity-Relationship (сущности и связи предметной области)  
**Основа:** MECE T1/T2, GQM T1 (`06_GQM_*`), FD T1–T3, `A_канон_диплом.md`  
**Дата:** 16.06.2026 · **Статус:** ✅ T1

**Назначение:** явно развести **объекты рынка** (бренд, оператор, импорт, СТМ) и **объекты измерения** (база S2/V1, снимок доли, p.p., маркировка), чтобы не смешивать их в тексте диплома — дополнение к GQM Anti-metric.

---

## 1. Логика ER для DR-A

| Слой | Сущности | Роль |
|------|----------|------|
| **L1 Рынок** | `Market`, `ProductSegment`, `GeoProxy` | Граница объекта |
| **L2 Игроки** | `Brand`, `Operator`, `Importer`, `STMCase` | Кто на полке / в канале |
| **L3 Измерения** | `MetricBase`, `ShareSnapshot`, `DeltaPP`, `MarketCapacity`, `ImportFlow`, `MarkingStat` | Цифры и факты |
| **L4 События** | `CorporateExit`, `PriceShock`, `RegulatoryChange` | Причины сдвига |
| **L5 Норма** | `ComplianceContour`, `ParallelImportRegime` | §3.9 |
| **L6 Источники** | `SourceID` | S2-01, V1-01, CZ-01… |

**Правило:** связь **Brand ↔ ShareSnapshot** всегда через **MetricBase**; **ImportFlow** не связывается напрямую с **ShareSnapshot** без пометки «≠».

## 5. Критические разрывы (D3, R8, R16) — «не связывать напрямую»

```
┌─────────────┐     exit 2022      ┌─────────────┐
│  Operator   │ ─────────────────► │   Market    │
│  (Shell)    │                    │  (official) │
└─────────────┘                    └─────────────┘
       │                                   │
       │ ≠ исчезновение                    │
       ▼                                   ▼
┌─────────────┐   parallel + import  ┌─────────────┐
│   Brand     │ ◄─────────────────── │ ImportFlow  │
│  (Shell)    │                      │  (AS-03)    │
└─────────────┘                      └─────────────┘
       │
       │ has_snapshot (S2 2023: 4,5% л)
       ▼
┌─────────────┐
│ShareSnapshot│  ≠  ImportFlow volume share
└─────────────┘
```

**Тезис D3 (ER):** `Operator` — `exited` — `Market`, но `Brand` — `persists_after` — `CorporateExit` и `Brand` — `supplied_via` — `ImportFlow`.
