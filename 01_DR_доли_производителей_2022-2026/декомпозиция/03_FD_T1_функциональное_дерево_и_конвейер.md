---

## 1. Логика инструмента

MECE ответил **«что где лежит»** (блоки A–G). Functional Decomposition отвечает **«что с этим делать»**: какие **функции** выполняются над каждым блоком, в каком **порядке**, какой **выход** идёт в главу диплома.

Универсальная цепочка для DR-A:

```
ПОЛУЧИТЬ → ПРОВЕРИТЬ → СТРУКТУРИРОВАТЬ → ОФОРМИТЬ → ИНТЕГРИРОВАТЬ
(F1)       (F2)         (F3)              (F4)        (F5)
```

- **F1 Получить** — извлечь факт из run1/run2 + URL-реестра  
- **F2 Проверить** — tier, 🚩, база (S2/V1), GATE, R1–R18  
- **F3 Структурировать** — таблица, тезис, MECE-блок  
- **F4 Оформить** — абзац/§ диплома + сноска (ID источника)  
- **F5 Интегрировать** — связь с СТМ/GTM, wiki, канон

## 6. Сквозной конвейер (критический путь)

Приоритет из T2 — **минимальный набор функций для закрытия главы**:

```
[1] C1: F1→F2→F3→F4  →  §3.3 таблица S2 (литры)           ★ критический
[2] D0+D4: F1→F3→F4  →  §3.4 абзац динамики + V1 p.p.    ★ критический
[3] A3: F2→F4        →  §3.1 двухслойная методология     ★ критический
[4] F0+F1: F1→F4     →  §3.9 compliance                   ○ важный
[5] D2.1: F1→F4      →  §3.5 кейс Lemarc (качественно)    ○ важный
[6] G1–G5: F4→F5     →  §4 импликации СТМ                ○ важный
[7] G6–G7: F4        →  приложение «метрики запуска»     △ опционально
```

**Параллельные ветки** (не блокируют [1]–[3]): B, E, D5–D8, A2.
