# План диплома: собственная торговая марка автомасел (этапы 0–7)

> **Статус:** план **согласован с руководителем** (27.05.2026). Эталон встречи: `план_СТМ_1_слайд.pptx`. Протокол: `диплом_СТМ_автомасла/00_декомпозиция_и_дорожная_карта/согласование_с_руководителем.md`.  
> **Текущий фокус:** этап **1** (мета-промпты DR).

**Тема:** проектирование и обоснование запуска **собственной торговой марки (СТМ)** легкомоторных масел (PVL).  
**География:** **РФ, европейская часть** (спрос, каналы, референсы по полке/e-com, климат). **Нормативка:** **РФ / ЕАЭС**.  
**Оговорка по данным:** при использовании **федеральных** отчётов о долях — явная пометка и переносимость выводов на европейскую часть, если нет регионального среза.

**Опоры:** `Памятка_маршрутизация_руководств.md`, `Памятка_промпты_ОЕ_и_руководства.md`, `Памятка_конспекты_Нейромаркетинг_и_ИИ_в_маркетинге.md`.  
**Журнал работ:** `диплом_СТМ_автомасла/журнал_проекта.md` (задачи, артефакты, руководства, промпты, отправки преподавателю).

**Мета-промптер Oracle:** [GPToracle — The AI Research Meta Prompter](https://chatgpt.com/g/g-67cf7a6be27c8191835abbb61a8f9bed-gptoracle-the-ai-research-meta-prompter)

## 1. Логика этапов 0–7

| Этап | Название | Суть | Ключевой артефакт |
|:----:|----------|------|-------------------|
| **0** | Декомпозиция | Дорожная карта, критерии успеха, структура папок | План работ + рубрика оценки |
| **1** | Мета-промпт DR | Постановка исследований через Oracle + протокол аналитика | 2–3 шаблона промптов для DR |

| **3** | DR: вязкости | Спрос по маркировке (SAE и смежные рамки) | Разрезы + ограничения данных |
| **4** | Упаковка конкурентов | **PGMM + ODSA** по этикеткам/канистрам | Сравнительная матрица брендов |

**Этапы 1–6** — эмпирическое и проектное ядро. **Этапы 0 и 7** — рамка проекта и защищаемый итог. Без этапа 7 диплом рискует выглядеть как набор отчётов LLM без единой стратегии СТМ.

## 2. Протокол агента-аналитика: как применяется

По **`Протокол_Агента_Аналитика_Спецификация_и_Синтез_Задач.pdf`** на **каждый крупный артефакт** (не один раз «на весь диплом»):

1. **Таксономия** — цель, контекст, получатель, материал, критерии (**1.x**), ограничения (**1.w**), формат (**1.y**), алгоритм (**1.t**).
2. **Синтез** — финальный «самораспаковывающийся» промпт для исполнителя.
3. **Исполнение** — новый чат, файл руководства + промпт.
4. **Ревизия** — **Red Team / Blue Team** (цепочка из `Промпты ОЕ/Протокол_Агента_Аналитика…`).

**Минимум прогонов протокола:**

| Код | Артефакт | Этап |
|-----|----------|:----:|
| A | DR: доли рынка | 2 |
| B | DR: вязкости | 3 |
| C | PGMM + ODSA по конкурентам | 4 |
| D | Проверка названия | 5 |
| E | Бриф на визуал (по желанию) | 6 |

**Схема потока:**

```
[0] Декомпозиция
         ↓
[1] Промпты DR (Oracle)  ←── протокол аналитика
         ↓
[2] DR доли  ──┐
[3] DR вязкости├─→ Декомпозиция (MECE, Ishikawa, Pareto) / Red–Blue на каждый отчёт
[4] PGMM+ODSA ─┤
[5] Название ──┤
[6] Эскиз ─────┘
         ↓
[7] Синтез → текст диплома + приложения
```

## Этап 1. Мета-промпт для Deep Research (GPToracle)

**Цель:** получить 2–3 **отдельных** промпта для DR (не один «про всё»).

**Вход для Oracle (через протокол аналитика):** география (**РФ, европейская часть**), источники, единица доли, годы **2022–2026**, бренды (Mobil, Shell, Castrol, Elf, Total, Gazpromneft, LUKOIL, Rosneft, ZIC и др.), решения по итогам, формат отчёта.

| Действие | Руководство |
|----------|-------------|
| Сбор таксономии | Протокол аналитика → 1.j, 1.x, 1.w, 1.y |
| Сборка промпта | GPToracle |
| Ревизия черновика | Red Team; для фактов — низкая температура |

**Шаблоны DR:** (A) доли производителей; (B) вязкости/маркировка; (C) опционально — каналы/сегмент СТМ.

**Протокол аналитика:** отдельный прогон на каждый шаблон A/B/(C).

**Артефакты:** `00_таксономии_и_промпты_аналитика/` — промпты, логи протокола, версии после Red/Blue.

## Этап 2. Deep Research: доли рынка 2022–2026

**Локус в промпте:** РФ, европейская часть; каналы (розница, СТО, маркетплейсы); импорт vs отечественные; **СТМ сетей** отдельно от брендов производителей; методология доли.

| Действие | Руководство |
|----------|-------------|
| Исполнение DR | Промпт из этапа 1A; ChatGPT DR / Gemini DR |
| Структура рынка | `Углублённое исследование_рынка_нишы_сегмента…xlsx` + нейромаркетинговый системный промпт |
| Проверка цифр и выводов | Декомпозиция (MECE, Ishikawa, Pareto) + `ODSA_Audit_Packet…md` |
| Противоречия источников | `DDx-6D_8D.pdf` |

**Протокол аналитика:** прогон **A** (до DR) + ревизия результата (Red/Blue).

**Артефакты:** `01_DR_доли_производителей_2022-2026/` — отчёт, таблицы, `literature_и_источники.md` (фрагмент).

## Этап 3. Deep Research: спрос по вязкостям

**Принцип:** **1 DR ≈ 1 тема**. Не смешивать «вязкость» и «тип/класс масла». Контекст: **SAE J300**, ACEA, OEM-допуски; сезонность для умеренного климата европейской РФ.

| Действие | Руководство |
|----------|-------------|
| Исполнение DR | Промпт из этапа 1B |
| Проверка | Декомпозиция (MECE, Ishikawa, Pareto) + ODSA + перекрёстная сверка (Perplexity и т.д.) |

**Протокол аналитика:** прогон **B** + ревизия.

**Артефакты:** `02_DR_вязкости_спрос/` — разрезы (0W-20, 5W-30, 5W-40…), ограничения данных.

## Этап 4. Упаковка и этикетка конкурентов (PGMM + ODSA)

**Правило:** графика, текст и управляющие элементы — **раздельные** проходы. Один бренд = одна сессия. SKU для **витрины РФ (европейская часть)**.

| Действие | Руководство |
|----------|-------------|
| Разбор по PGMM (`_full`) | `PGMM.pdf` + `Исследование продукта… PGMM 13.03.26.xlsx` |
| Пошаговый аудит (gate-одобрение) | По аналогии с `Исследование страницы… PGMM 20.03.26.xlsx` |
| Claims на этикетке | `ODSA_Audit_Packet…md` |
| Сводная матрица | Markdown/таблица; при спорах — DDx |

**Вход:** фото высокого разрешения; OCR/ручной перенос мелкого текста.

**Протокол аналитика:** прогон **C** (на блок конкурентов) + ревизия матрицы.

**Артефакты:** `03_PGMM_ODSA_упаковка_конкуренты/`.

## Этап 6. Эскиз упаковки СТМ (графические ИИ)

**Вход:** выводы этапов **4** (PGMM) и **5** (имя). Обратная постановка: домены A/B → бриф по блокам, не «нарисуй красиво».

| Действие | Руководство |
|----------|-------------|
| Бриф из PGMM | `PGMM.pdf` (PROC/CASE) |
| Генерация | `Промпт-хаки для Nano Banana…pdf` |
| Снижение шаблонности | Несколько прогонов, свои PGMM-референсы |
| Макет и маркировка | ODSA по мокапу (язык, обязательные поля **РФ**) |

**Протокол аналитика:** прогон **E** — по желанию, если бриф сложный.

**Артефакты:** `05_визуальные_эскизы/` — мудборд, 3–5 вариантов, таблица «элемент → обоснование».

| Методология | Протокол аналитика, ODSA, DDx, PGMM, DR | Протокол + памятки |
| Риски и допущения | Доли рынка, регион, LLM | `допущения_и_риски_методологии.md` |
