## Правила обновления (Ingest)

После **декомпозиции DR** (2a/3a/…), **нового отчёта DR**, **верификации URL** или **согласования с руководителем**:

1. Обновить **одну** тематическую страницу `wiki/XX_….md` (не дублировать в корень проекта).
2. Добавить запись в `wiki/log.md` (дата, что изменилось, источник).
3. При новой странице — строка в `wiki/index.md`.
4. Обновить `wiki/overview.md`, если меняется общий вывод.
5. **Не** копировать целиком отчёты DR в `wiki/` — только выжимка + ссылки на `raw/`.

## Жёсткие ограничения (диплом)

- **DR-A и DR-B раздельно:** не смешивать таблицы брендов и SAE.
- **Две базы DR-A:** S2 (retail 2023) ≠ V1 (aftermarket 11М2023) — не в одной таблице.
- **2024–2026 доли брендов:** н/д в open data — не интерполировать.
- **DR-B:** без минералки; европ. часть = ЦФО+СЗФО+ЮФО; часто **федеральный прокси**.
- **URL в сносках:** только из `04_источники_и_URL.md` / реестров; без `utm_source=chatgpt.com`.
- **Kolesa:** slug `rastushhem`, не `rastushchem`.
- **Фото vs канон specs (PGMM/ODSA):** фото канистр из интернета часто **устаревшие**. При конфликте поколений (**API SP** vs **SN**, ACEA C2/C3 vs C3-only и т.п.) — **канон SKU = наивысший допуск** по иерархии (API: **SP > SN > SL**…; OEM/ACEA — полный набор с **официального сайта** оператора). Digital triangulation **перекрывает** legacy-facing на фото. В отчётах: **artifact** (что на фото) и **canonical SKU** (строка матрицы) — **раздельно**.

## Workflow: Query

На вопрос пользователя:

1. index → релевантные страницы.
2. Ответ **со ссылками на wiki-страницы** и ID источников (S2-01, BX-02…).
3. Пометка **н/д** / **прокси** / **tier**, если данные слабые.
4. Не выдумывать цифры — только из вики или указанного raw-файла.

---

## Workflow: Skills (обязательно)

Когда этап **существенно проработан** (DR-run, декомпозиция, ingest wiki, согласование):

1. **Предложить пользователю** skill: имя, зачем, как вызвать (`используй skill <name>`).
2. После «да» / «создай» — `.cursor/skills/<name>/SKILL.md` + строка в `wiki/09_skills_реестр.md`.
3. **Не создавать молча** — всегда явное предложение в конце работы по этапу.

Реестр: `wiki/09_skills_реестр.md`. Уже есть: `stm-diploma-wiki`, `stm-dr-decompose-a`, `stm-dr-decompose-b`, `stm-dr-prompt-pipeline`, `stm-pgmm`, `stm-pgmm-delta`, `stm-odsa`.
