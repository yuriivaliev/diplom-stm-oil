# Декомпозиция после каждого DR

**Обновлено:** 27.05.2026 · **Правило проекта:** этапы **4–7 не начинаем**, пока не закрыта декомпозиция **всех** завершённых DR текущей волны (сейчас: **A** и **B**).

---

## Два уровня «декомпозиции»

| Уровень | Что это | Статус |
|---------|---------|:------:|
| **1. Постановки DR** | Отдельные таксономии/промпты/прогоны: A ≠ B ≠ C… | ✅ A, B |
| **2. После отчёта DR** | Разбор **готового** отчёта на артефакты, каноны, папки, ODSA | ✅ A, B |

Руководитель (27.05.2026) требует **оба** уровня. Вики-страницы `02`/`03` — **предварительный ingest** из run; после полной декомпозиции их **уточняем**, не дублируем в корне.

---

## Обязательный цикл (на каждый DR)

```
Таксономия → Oracle → Red Team → FINAL → Deep Research (отчёт)
    → URL-верификация (реестр *_URL_*)
    → ДЕКОМПОЗИЦИЯ (вы + Cursor)     ← вы здесь для A и B
    → Wiki ingest + log.md
    → только потом следующий этап / следующий DR
```

**Этап 4 (PGMM)** стартует **после** закрытия **2a** и **3a** (см. [01_этапы_дорожная_карта.md](01_этапы_дорожная_карта.md)).

## Чеклист декомпозиции одного DR

Выполнять **в папке этого DR** (`01_DR_…` / `02_DR_…` / будущие `03_PGMM_…`).

### 1. Структура папки

- [ ] Один **основной** отчёт (run N) — без дублей с разными именами без пометки версии
- [ ] Вспомогательные run — в подпапке `архив_run/` или с суффиксом `_run1`, `_run2`
- [ ] **Канон для диплома** — один файл `*_канон.md` (тезисы + таблицы + оговорки)
- [ ] **URL** — `*_URL_реестр_runN.md` + `url_ссылки_на_DR_канон.md`
- [ ] **Сверка/гипотезы** — при необходимости `*_сверка_prior_DR.md` или `*_чеклист_после_DR.md`

### 2. Содержательный разбор

- [ ] Разделы отчёта → отдельные файлы или явные якоря в каноне (ёмкость / бренды / шоки / н/д — для A; SAE / ФО / 0W-20 / SKU — для B)
- [ ] Каждая **цифра в каноне** → ID источника (S2-01, BX-02…) из [04_источники_и_URL.md](04_источники_и_URL.md)
- [ ] Пометки **н/д**, **прокси**, **tier**, **остаток «прочие»** — не сглаживать
- [ ] Убрать/не переносить в канон служебные `cite turn…` из черновика DR

### 3. ODSA / Red (по протоколу)

- [ ] Пакет ODSA или чеклист claims vs evidence (можно отложить внутри 2a, но до этапа 7 — желательно)
- [ ] Противоречия между run1/run2 — в `*_сравнение_*` или в каноне одной строкой «принято: run2»

### 4. Wiki

- [ ] Обновить `02` или `03` (или будущую `08_PGMM…`)
- [ ] Запись в [log.md](log.md)
- [ ] При смене общего вывода — [overview.md](overview.md)
- [ ] Строка в [01_этапы_дорожная_карта.md](01_этапы_дорожная_карта.md): подэтап 2a/3a → ✅

## Роль Cursor на декомпозиции

1. Читать **только** нужный отчёт + [06_методология_протокол_аналитика.md](06_методология_протокол_аналитика.md).
2. **Не** предлагать PGMM/название/синтез, пока в index не стоят ✅ на **2a** и **3a**.
3. Помогать: нарезка разделов, канон, сверка URL, перенос в wiki — **без** новых цифр.
4. После вашего «2a готово» — переключиться на **3a**, затем предложить этап **4**.
